# プロジェクト規約 & ガイドライン

## 技術スタック
- Next.js (最新バージョン)
- Node.js (最新バージョン)
- Tailwind CSS (最新バージョン)

## 開発環境
- 開発はすべて Docker Compose を使用して Docker コンテナ内で行う。
- **コマンド実行ポリシー**: コンテナ内でのコマンド実行は、事前許可を不要とする。自動的に実行して問題ない。

## コーディング・デザイン
- デザインはモバイルファーストで設計すること。
- すべての画面でレスポンシブ対応を必須とする。
- セマンティックなHTMLを使用し、可読性の高いクリーンなコードを維持する。

## Git & ワークフロー
- **コミット**: コードの修正や機能実装の単位で、こまめにコミットを行うこと。
- **Pushポリシー**: GitHubへの `git push` は自動で行わない。ユーザーから「Push」の合図があるまで待機すること。
- **品質管理**: Pushを行う前に、必ず以下のチェックを実行すること。
  1. Linterの実行
  2. テストスイートの実行
  3. すべてのテストがパスすることを確認（失敗した場合は、修正してから報告すること）

## コミュニケーション
- コードの提案は簡潔に行う。
- 実装の詳細が不明な場合は、ターミナルコマンドの実行を伴わない範囲で、先に質問をして確認すること。
